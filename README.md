<h1>Relatório</h1>
<p>Para conseguir fazer todas as remoções dos nós de uma árvore binária, eu criei 7 funções (tanto iterativas quanto recursivas): </p>
<h2>remover</h2>
<p>É uma função de validação (se a árvore está vazia e se o elemento passado é nulo) que chama a função procurarElemento. Como ela é apenas uma função de validação para passar os elementos para a função procurarElemento, ela não possui laços de repetição nem recursividade.</p>
<h2>procurarElemento</h2>
<p>É a função que percorre toda a árvore em busca do elemento a ser removido. Ela verifica se o elemento existe na árvore e compara os conteúdos dos nós com o elemento dado. Caso o elemento condiga com o conteúdo do nó atual, a função tipo é chamada. Caso o elemento seja diferente do conteúdo do nó atual, a função verifica se o elemento passado é maior ou menor que a raiz. Se for maior, a função procurarElemento é recursivamente chamada, recebendo como parâmetros o elemento passado, o nó à direita do atual e o nó atual. Caso contrário, a função procurarElemento é chamada, mas dessa vez recebendo como parâmetros o elemento passado, o nó à esquerda do nó atual e o nó atual. Essa função foi feita de forma recursiva para facilitar o processo de busca e para reutilização de código. O processo de busca dos elementos, tanto à esquerda quanto à direita, seria o mesmo processo que já havia sido feito, mas dessa vez recebendo os nós ou à direita ou à esquerda, dependendo do valor deles comparado com a raiz.</p>
<h2>tipo</h2>
<p>É a função que chama as funções de remoção. Ela compara cada nó para descobrir que tipo de remoção deve ser feita. Se o nó não possuir nós nem à direita nem à esquerda, é um nó folha (pode ser apenas um nó folha ou uma raiz sem filhos), ou seja, a função removerNoFolha é chamada. Se o nó possui um filho à direita mas não possui nenhum à esquerda, é um nó com filho à direita (pode ser tanto um nó filho ou uma raiz com apenas um filho à direita), logo, a função removerNoFilhoDireita é chamada. Se o nó possui um filho à esquerda mas não possui nenhum à direita, é um nó com filho à esquerda (pode ser tanto um nó filho ou uma raiz com apenas um filho à esquerda), logo, a função removerNoFilhoEsquerda é chamada. Por fim, se o nó possui filho tanto à esquerda quanto à direita, ele é um nó com dois filhos (pode ser um nó filho ou uma raiz com dois filhos), então a função removerNoDoisFilhos é chamada. Essa função é apenas uma função de validação e redirecionamento, logo, não possui laços de repetição nem recursividade.</p>
<h2>removerFolha</h2>
<p>É a função chamada pela função tipo para remover os nós folhas. Ela verifica se o nó folha é a raiz e, se não for, se o nó está do lado esquerdo ou direito da raiz. Essa função faz a remoção a partir do pai do nó folha e não possui repetições, logo não é iterativa e não poderia ser recursiva.</p>
<h2>removerNoFilhoDireita | removerNoFilhoEsquerda</h2>
<p>A função removerNoFilhoDireita é a função chamada pela função tipo para remover os nós que possuem filhos do lado direito. Já a função removerNoFilhoEsquerda é a função chamada pela função tipo para remover os nós que possuem filhos do lado esquerdo. Elas seguem a mesma lógica da função removerFolha e também não são recursivas nem iterativas.</p>
<h2>removerNoDoisFilhos</h2>
<p>É a função chamada pela função tipo para remover os nós que possuem dois filhos. Essa função encontra o menor dos maiores utilizando um laço de repetição while, seta o valor a ser eliminado como o menor dos maiores e chama a função tipo novamente, utilizando as outras funções de remoção para remover o nó que ficou repetido e, caso esse nó tinha um filho à direita, organiza esse filho na árvore. Essa função foi implementada de forma iterativa, utilizando um while para ir continuamente para a esquerda e encontrar o menor dos maiores.</p>
